四、涉及文件变更
文件变更内容
MainScreen.kt 1. 移除全屏遮罩的 workRunning 逻辑
2. Header 增加 HeaderPill 组件（三种状态）
3. 移除底部索引按钮
4. 首次打开 DB=0 时显示索引选择页
5. 再次打开时判断并弹窗
6. 索引页改为可退出页面（不再是全屏遮罩）
MainViewModel.kt 1. 新增 totalPhotoCount: StateFlow 首次扫描获取 MediaStore 总数
2. 新增 isIndexComplete: StateFlow 判断 DB count == total
3. 新增 continueIndexing() 方法判断入队哪个 Worker
4. 新增 hasShownResumeDialog: MutableStateFlow 控制弹窗只显示一次
IndexProgressView.kt 改为可退出的独立页面，增加返回按钮
HeaderIndexPill.kt (新) 新建 Header Pill 组件：idle / indexing / done 三种状态
IndexChoicePage.kt (新) 新建索引选择页面：快速 Top 100 / 索引全部 / 显示总数
关键设计决策：
1. 索引完全依赖 App 运行 — 关闭 App 即取消 Worker
2. 首次扫描获取 MediaStore 总数（轻量 COUNT 查询）
3. Header Pill 三种状态：紫色「开始索引」、黑色呼吸点「N/总数」、绿色「已完成」
4. 底部不再保留任何索引相关按钮 — 简洁搜索页

# GPS 定位方案文档

> 本文档记录 PicSearch App 如何从手机照片中提取地理位置信息，并将其转换为可读的地址（如「北京市朝阳区」）。

---

## 一、我们是怎么获取照片 GPS 的

### 基本原理

手机拍照时，相机 App 会把当时的 GPS 坐标写入照片的 EXIF 信息中。我们的 App 在索引照片时读取这些 GPS 坐标。

### Android 10+ 的隐私保护机制

从 Android 10 开始，系统出于隐私保护，**默认会移除照片中的 GPS 信息**。即使 App 有读取照片的权限，拿到的也是「净化后」的副本，GPS 被悄悄删除了。

要获取原始 GPS，必须同时满足两个条件：

1. **声明并获取 `ACCESS_MEDIA_LOCATION` 权限**
   - 在 AndroidManifest.xml 中声明
   - 用户点击"开始索引"时动态弹出授权弹窗
   - Android 10 及以上需要此权限

2. **调用 `MediaStore.setRequireOriginal()`**
   - 告诉系统：我需要原始文件，不要给我净化副本
   - 只有调用后，系统才会返回包含 GPS 信息的文件
   - Android 11+ 直接调用该方法，Android 10 通过 Bundle 传参实现相同效果

### 完整读取流程

```
用户点击"开始索引"
    │
    ├─ 请求照片权限 + 位置权限
    │
    ├─ 用户授权
    │
    ├─ 遍历手机中的照片
    │   │
    │   ├─ 主路径：通过 setRequireOriginal 标记的 URI 查询
    │   │         MediaStore 的 LATITUDE/LONGITUDE 列，直接拿到 GPS
    │   │
    │   └─ 备用路径：如果上面拿不到，就用原始 URI 打开文件，
    │               直接读取照片 EXIF 中的 GPS 标签
    │
    └─ GPS 存入本地数据库
```

## 二、遇到的问题

### 问题：GPS 始终为 0, 0

**表现**：索引照片后，数据库中所有图片的经纬度都是 null / 0。

**根因**：Android 10+ 系统自动剥离了 EXIF 中的 GPS 数据。我们之前虽然修复了 EXIF 读取方式（用 FileDescriptor 替代 InputStream），但文件本身就不包含 GPS 信息了，所以无论怎么读都是空的。

**解决**：
- 添加 `ACCESS_MEDIA_LOCATION` 权限
- 调用 `setRequireOriginal()` 获取原始文件
- 两个条件缺一不可

### 其他可能影响 GPS 获取的情况

- **图片本身没有 GPS**：部分 App 拍的照片或从微信/微博保存的图片，原始文件就不含 GPS 信息
- **用户拒绝授权**：如果用户不给位置权限，GPS 数据将无法读取
- **Android 9 及以下**：不受隐私机制限制，直接读取即可

## 三、GPS 坐标 → 可读地址

### 当前方案：市级精度

拿到 GPS 坐标（如 `39.9042, 116.4074`）后，用户看到的是数字，不直观。我们需要转换成「北京市朝阳区」这样的文字地址。

**实现方式**：

1. 在 App 中内置两个轻量级数据文件（放在 assets 目录）：
   - **全球国家边界**（~80 KB）— 来自 Natural Earth 开源数据
   - **中国地级市边界**（~500 KB）— 来自 modood/Administrative-divisions-of-China（民政部权威数据）

2. 判断逻辑：
   - 先判断是否在中国境内
   - 如果在中国，匹配到对应的地级市，显示「XX 市」
   - 如果在国外，匹配到对应国家，显示国家名称（如「日本」）
   - 都匹配不到，显示「未知位置」（海洋、极地等）

**精度说明**：
- 中国境内：精确到地级市（如「北京市」「上海市」「广州市」）
- 中国境外：精确到国家（如「日本」「美国」）
- APK 增加体积：约 **600 KB**

### 如果需要区县级精度

如果未来想把精度提升到区县级（如「北京市朝阳区」），需要：

1. **替换数据源**：使用 modood 项目中的区县级 GeoJSON 文件（districts.json）
2. **体积变化**：APK 体积会增加约 **3~5 MB**（压缩后）
3. **代码改动**：查询逻辑基本不变，只是从「市级数据文件」换成「区县级数据文件」
4. **精度提升**：从市级（~300 个）到区县级（~2800 个），能精确到具体的区县

**简单对比**：

| 精度 | 数据文件 | APK 增加体积 | 示例 |
|------|----------|-------------|------|
| 市级（当前） | cities.json | ~600 KB | 北京市 |
| 区县级 | districts.json | ~3~5 MB | 北京市朝阳区 |

### 为什么不使用在线地图 API

Google Maps、高德地图等都提供在线反向地理编码 API，能把 GPS 转成详细地址。但我们选择离线方案的原因：

- PicSearch 的定位是**纯离线运行**，不依赖网络
- 离线方案保护用户隐私，位置信息不会上传到任何服务器
- 市级精度只需要 ~600 KB 数据，对 APK 体积影响极小
